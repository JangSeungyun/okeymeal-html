# 🏁 경쟁 서비스 벤치마크 (Competitor Benchmark)

OkeyMeal MVP(외국인 관광객 식이 케어, 제주도)의 시장 포지셔닝과 차별화 가설을 검증하기 위해 종합 맛집·관광 음식 3종 + 식이 제약 특화 2종 = **5개 서비스**를 7개 축으로 비교 분석합니다.

> 📌 **본 문서의 위치와 정책**
> - 본 문서는 **WBS Task 1.5**의 산출물로, M1 게이트(2026-05-19) 통과 게이트 자료입니다.
> - **시장 공백 가설**(메뉴 단위 ISO 알레르겐 매핑 + 다국어 OCR + 점주 콘솔의 결합)을 1차 검증하며, 최종 검증은 Phase 2 PoC 시점에 재조사합니다.
> - 일부 항목은 사전 지식 기반(2026-01 컷오프)이며, §6 "제한 사항 및 후속 검증" 표에 명시한 4개 항목은 Phase 2 W3 시작 전 재검증합니다.

## 3. 서비스별 상세 분석

### 3.1 C1. 망고플레이트 (한국 종합 맛집)

- **다국어**: 한국어 본판 + 영어 앱 별도. 일·중 공식 미지원. 리뷰는 한국어 중심.
- **식이 필터**: 알레르기·비건·할랄 전용 필터 **없음**. 키워드 검색 수준.
- **메뉴**: 사용자 업로드 사진 중심, 메뉴별 알레르기 라벨링 없음.
- **추천**: TOP100·랭킹·평점 기반. 위치·카테고리 필터 제공, 개인화 비중 낮음.
- **타겟팅**: 외국인 결제 장벽은 낮으나 영문 콘텐츠 양·최신성 부족. 2022년 이후 운영 축소·서비스 변동 이슈.
- **시사점**: 한국 종합 맛집 시장의 영문 콘텐츠 **공급 공백**을 제주 인바운드 관광객 대상으로 흡수 가능.

### 3.2 C2. Visit Korea Food (한국관광공사 공식)

- **다국어**: 10+ 언어, **사람 번역 기반** 공식 콘텐츠.
- **식이 필터**: 음식 카테고리 중심, 단 **할랄 식당 분류 가이드**(Self-Certified / Halal Certified / Muslim Friendly / Pork Free)를 별도 운영(아미나 페르소나 직접 관련).
- **메뉴**: 대표 K-Food 소개 기사·역사·재료. 식당 단위 실시간 메뉴 없음.
- **추천**: 에디토리얼 큐레이션(테마·시즌·지역). 알고리즘 개인화 없음.
- **타겟팅**: 외국인 1차 채널, 결제·예약 없음(정보형). 1330 다국어 콜센터 연계.
- **시사점**: **신뢰 가능한 에디토리얼**(공공)을 OkeyMeal의 Cold-start 초기 큐레이션 셋으로 활용 가능(공공 데이터 라이선스 확인 필요). 다만 OkeyMeal은 실시간 메뉴·식당 단위 데이터라는 점에서 포지셔닝이 보완 관계.

### 3.3 C3. Yelp (글로벌)

- **다국어**: 30+ 언어 UI, 구글 자동 번역 보조.
- **식이 필터**: Vegan/Vegetarian/Gluten-Free/Halal/Kosher 등 **카테고리 속성** 제공. 단 식당 단위, 메뉴 단위 없음.
- **메뉴**: PDF/텍스트 업로드, SinglePlatform 연동(일부). 자동 알레르겐 라벨링 없음.
- **추천**: Yelp Sort(평점·리뷰 수·근접도 + 신뢰도 가중). 명시적 개인화 엔진은 약함.
- **타겟팅**: 한국 서비스 사실상 **운영 중단/축소**(yelp.co.kr 비활성, 확인 필요). 한국 식당 데이터 커버리지 매우 낮음.
- **시사점**: 한국 시장에서 **공급 공백** 명확. 단 광고 의존 비즈니스 모델은 검색 신뢰도 훼손 리스크를 시사 — OkeyMeal은 초기에 점주 광고 의존을 회피해야 함.

### 3.4 C4. HappyCow (글로벌 비건·할랄 디렉터리)

- **다국어**: 영어 중심 + 유럽어 일부. **한·일·중 UI 미흡**(확인 필요).
- **식이 필터**: Vegan/Vegetarian/Veg-options 3단계 + Gluten-free/Halal/Kosher/Raw 등. **개별 알레르겐(8대/14대) 세분 필터 없음**, 라이프스타일 분류에 가까움.
- **메뉴**: 사용자/점주 업로드 사진·텍스트. **OCR·다국어 번역 없음**, 알레르겐 라벨링 없음.
- **추천**: 위치 + 별점 + Top Rated/Most Reviewed 단순 소팅. ML 개인화 제한적.
- **타겟팅**: 글로벌 비건 여행자가 핵심. City Guides·Trip Planner 큐레이션. 결제·예약 없음. 한·일·중 UI 미흡은 동아시아 관광객 흡수 장벽.
- **시사점**: 페르소나 B(아미나·할랄) 핵심 경쟁자. OkeyMeal은 **메뉴 단위 OCR + ja/zh-CN UI**로 차별화 가능. 단 **커뮤니티 리뷰·사진**이 신뢰 형성의 핵심이라는 점은 OkeyMeal도 초기 콘텐츠 부트스트랩 전략으로 채택 필요.

### 3.5 C5. Fig (알레르기 스캐너, 미국)

- **다국어**: 영어 단일. 한·일·중 미지원(확인 필요).
- **식이 필터**: **본 카테고리의 강점** — 50+ 식이 프로필(글루텐·유제품·견과·갑각·콩·달걀·비건·케토·저FODMAP·코셔 등) 사용자 정의. 스캔 결과 즉시 "Safe / Avoid" 판정. **EU 14대 / FDA 9대 ISO 알레르겐 코드 매핑 여부 불명**(확인 필요).
- **메뉴**: 식료품 바코드/라벨 스캔이 본업. **메뉴 사진 OCR은 일부 버전·베타 수준**(확인 필요).
- **추천**: 스캔 즉시 매칭이 메인. 위치 기반 식당 추천은 부차적.
- **타겟팅**: 미국 거주 알레르기 환자·가족이 주 타겟. **관광객 시나리오 거의 없음**, 영어 단일 UI.
- **시사점**: 페르소나 A(마크·견과 알레르기) 핵심 경쟁자. OkeyMeal은 **다국어 메뉴 OCR + 한국 식약처 데이터 + 관광객 동선 통합**으로 차별화. Fig의 **"Safe / Avoid 즉시 판정 UX"**(녹/적색 신호)는 OkeyMeal AI 렌즈 결과 화면에 직접 반영할 만한 UX 레퍼런스.

## 4. OkeyMeal 차별화 분석 (시장 공백 가설 1차 검증)

| 차별화 축 | 시장 현황 | OkeyMeal MVP | 검증 결과 |
|---|---|---|---|
| **A. AI 메뉴 OCR × ISO 알레르겐 × 다국어** | 세 축을 결합한 직접 경쟁자 사전 지식 기준 **없음** (HappyCow=라이프스타일 태그, Fig=식료품 라벨·영어). | INF-02·INF-04·USR-3.1 (AI 렌즈), ISO 코드 룰엔진 우선 + LLM 보조 | ✅ **공백 가설 유효** (Phase 2 W3 재검증 필수, §6 검증 항목 1·2·4) |
| **B. 점주 콘솔(다국어 메뉴 입력)** | 모든 경쟁자가 사용자/큐레이터 UGC 또는 점주 참여 자체 없음. 점주가 표준 알레르겐 코드로 다국어 메뉴를 등록하는 B2B 사이드는 **비어 있음**. | OWN-1.1·2.1~2.4 + `owner.okeymeal.kr` 서브도메인 + FCM Topic | ✅ **데이터 신뢰성 해자** — 제주 MVP 점주 온보딩이 정확도 결정 |
| **C. 동아시아(ko/ja/zh-CN) 관광객** | C1~C5 모두 영어/유럽어 중심. ja/zh-CN UI + OCR + 알레르겐 라벨링은 사각지대. | INF-03 다국어 3계층 + ko/en/ja/zh-CN 4종 | ✅ **제주 관광객 타겟 적합** (중국·일본 비중) |
| **D. 통합 동선 (탐색 + 안심 + 길찾기 + 점주 알림)** | 각 서비스가 하나의 축(맛집/큐레이션/식이/스캔)에만 특화. End-to-End 시나리오 통합은 없음. | USR-1~3 + OWN + 멀티 맵(`MapClient`) + FCM QR 안심 패스 | ✅ **시나리오 통합 우위** (페르소나 A·B·C 모두 단일 앱에서 완결) |
| **E. 비즈니스 모델 (관광객 무료 + B2B)** | Yelp 광고 의존 → 검색 신뢰도 훼손 리스크. Fig 구독 → 단기 관광객 ARPU 한계. | MVP 무료 + 향후 B2B(여행사·호텔 컨시어지·할랄/비건 인증 수수료) 가설 | ⚠️ **수익 모델 가설 단계** — Phase 2 종료 시 사업 모델 검증 별도 작업 필요 |

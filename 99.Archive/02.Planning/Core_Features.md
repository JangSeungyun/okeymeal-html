# 🛠️ 상세 기능 정의서 (Detailed Features)

OkeyMeal MVP의 기능 ID 체계와 우선순위를 정의하는 빠른 참조용 인덱스 문서입니다.

> 📌 **단일 진실 공급원(SSOT) 정책**
> - **수용 기준(AC)·사용자 스토리·의존성·WBS 매핑의 정본은** [`01.Ideation/Requirements.md`](../01.Ideation/Requirements.md) §7 (v0.1.4+).
> - **기술 결정(스택·아키텍처·운영)의 정본은** [`04.Design/Technical_Decision_Records.md`](../04.Design/Technical_Decision_Records.md) (v3.1.0+).
> - 본 문서는 ID 체계·기능명·1줄 설명·우선순위·핵심 AC 요약만 유지하며, **상세는 Requirements 링크를 따라간다**.
> - ID/기능명/우선순위 변경 시 Requirements §7·§14 추적성 매트릭스를 동시에 갱신해야 한다.

**우선순위 분류**
*   **P1**: 필수 구현 (MVP 핵심) | **P2**: 보완 기능 | **P3**: 확장 기능 / 본 MVP 제외

## 🏗️ 1. 시스템 공통 및 인프라 (System & Infra)

서비스의 뼈대가 되는 데이터 융합·AI·인증·알림 인프라입니다.

| 기능 ID | 기능명 | 상세 설명 | 우선순위 | 핵심 AC / Reference |
|---|---|---|---|---|
| INF-01 | **5-Layer 데이터 융합 엔진** | 관광공사(POI) + 식약처(성분/레시피) + 점주(직접 입력) + LLM 추론 + 의료기관 데이터를 결합하는 신뢰도 매핑 로직 | P1 | 출처 우선순위 **식약처 > 점주 > LLM**. L5(의료기관)는 SOS 호출 시점에만 결합. → Req §7.1 INF-01 |
| INF-02 | **RAG 기반 AI 추론 모듈** | LLM에 식약처/점주 컨텍스트를 강제 주입하고 `sources` 메타로 근거 추적 가능한 추론 응답 생성 | P1 | LLM Provider 추상화(EXT-09)로 워크로드별 모델 분리(PoC 후 Q1 확정). 신뢰도 60점 미만 시 면책 문구. → Req §7.1 INF-02 / TDR §1.5 |
| INF-03 | **다국어 통합 번역 레이어** | 4개 언어(ko/en/ja/zh-CN) 3계층(L1 정적 UI / L2 시스템 / L3 동적 데이터) 통합 i18n. L3는 JSONB로 사전 번역되어 LLM 호출 미발생 | P1 | 누락 시 ko 폴백 + 응답 메타 `fallback_locale`. → Req §7.1 INF-03 / TDR §4.9 |
| INF-04 | **이미지 OCR/Vision 처리** | 메뉴판/성분표 이미지에서 텍스트·객체 추출 파이프라인. ClamAV + MIME 매직넘버 검증 + 30분 자동 삭제 | P1 | OCR 엔진 선택(Q2): Cloud Vision vs Gemini Vision은 Phase 2 PoC 후 확정. P95 ≤ 8초. → Req §7.1 INF-04 / Req §16 Q2 |
| INF-05 | **실시간 알림 브릿지** | QR 스캔 → 점주 콘솔까지 **FCM 단일 채널**(Android·iOS·Web 통합)로 푸시 도달. Quiet Hours 22:00~08:00 큐잉 | P1 | P95 ≤ 5초. Topic `owner_{restaurantId}`. → Req §7.1 INF-05 / TDR §1.7 |
| INF-06 | **Auth & 보안 시스템** | Guest UUID 기반 **90일 JWT**(Zero Barrier) + 회원 OAuth2(Google·Apple·Kakao·Naver 4종) + Access 30분/Refresh 일반 30일·점주 7일 + 회전 재사용 탐지 | P1 | Guest → Member 전환 시 식이 프로필·찜 데이터 무손실 이관. → Req §7.1 INF-06 / TDR §1.5 |
| INF-07 | **메디컬 핫라인 커넥터** | SOS 호출 시 현 위치(±1km) 기반 외국어 진료 가능 응급실/약국 상위 3개 매핑. 외부 데이터(심평원/보건복지부) 다운 시 ±24h 캐시 폴백 | P1 | USR-3.4의 데이터 진입점. → Req §7.1 INF-07 |

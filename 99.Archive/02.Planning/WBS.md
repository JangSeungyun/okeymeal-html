# 📅 OkeyMeal WBS 초안 (Work Breakdown Structure)

본 문서는 OkeyMeal 프로젝트의 단계별 작업 분해 구조와 일정을 정의합니다. **2026년 9월 30일 정식 배포**를 데드라인으로 하며, 분석 → 설계 → 개발 → 테스트 → 배포 5단계로 진행합니다.

> 📌 **참조 문서**
> - 기술 결정: [`04.Design/Technical_Decision_Records.md`](../04.Design/Technical_Decision_Records.md)
> - 핵심 기능: [`02.Planning/Core_Features.md`](./Core_Features.md)
> - MVP 스코프: TDR §0 (예약·결제 제외, 점주 콘솔 서브도메인, AI 렌즈는 OCR 우선)

> 👥 **프로젝트 인력 (v0.2.0 기준, 2026-05-08 변경)**
> - **풀스택 시니어 2인 (15년차)** · 트랙 A(백엔드·AI·인프라) + 트랙 B(프론트·모바일·점주 콘솔)
> - 분업 상세 RACI: [`02.Planning/RACI.md`](./RACI.md)
> - 변경 사유: 1인 가정 → 시니어 2인 투입으로 §6 R6(개발 리소스 부족) 영향·확률 완화. 신규 리스크 R11(분업 인터페이스 정합)·R12(AI 전문성 갭) 추가.

## 5. 크리티컬 패스 및 의존성

```text
[M1 분석] ──→ [M2 설계] ──→ [M3 알파] ──→ [M4 베타] ──→ [M5 RC] ──→ [M7 배포]
                                  │              │
                                  ↓              ↓
                              MK1 계정 ────→ MK3 내부 ────→ MK5 심사 ──→ MK6 등록
```

### 가장 긴 의존 사슬 (Critical Path)
1. **백엔드 도메인 사슬:** ERD(2.1) → API 명세(2.3) → 도메인 구현(3.3) → AI 추천(3.5) → AI 렌즈(3.7) → 통합(3.11)
2. **AI 사슬:** AI 알고리즘 설계(2.2) → **Base Code PoC(2.12)** → 임베딩 배치(3.4.5) → 추천 엔진(3.5) → RAG 설명문(3.5.5)
3. **마켓 사슬:** M4 베타 → MK3 내부 → MK5 심사 → MK6 등록 (외부 의존)

### 병렬화 가능 영역
- (W2) 분석 잔여 task(1.1~1.6) ∥ Branch Protection 설정(1.7) — 트랙 간 무관
- (W3-W4) 알고리즘 설계(2.2)와 Base Code PoC(2.12)는 트랙 A 단독 사슬이므로, 트랙 B는 와이어프레임(2.4)·UI 시안(2.5)·점주 콘솔 설계(2.6)에 집중
- 인증 시스템(3.2)과 인프라 셋업(3.1)은 W7-W8에서 병렬 진행
- AI 추천 엔진(3.5)과 다국어 사전 번역(3.6)은 임베딩 배치(3.4.5) 완료 후 병렬
- 점주 콘솔(3.9)과 푸시 알림(3.10)은 W17에서 병렬

## 7. 진척 관리

### 주기적 회고
- **매주 일요일:** 주간 회고 (1시간) — 진척, 블로커, 다음주 계획
- **마일스톤 도달 시:** 게이트 점검 회고 (M1~M7)
- **회고 결과:** 본 문서 갱신 + `00.Meetings/`에 회의록 적재

### 본 문서 갱신 정책
- **Minor 갱신:** 작업 완료/지연 발생 시 해당 항목에 `✅ 완료 (날짜)` 또는 `⚠️ 지연 (사유)` 마킹
- **Major 갱신:** 마일스톤 도달 또는 일정 재조정 시 버전 업 (v0.2.0 → v0.3.0)
- **연동 문서:** TDR(`04.Design/Technical_Decision_Records.md`), Core_Features 변경 시 본 문서 영향 검토
- **🔁 WBS.html 동기화 의무 (필수):** 본 문서(`WBS.md`)를 수정할 경우 **반드시 같은 커밋에서 [`WBS.html`](./WBS.html)도 함께 갱신**합니다. 시각화 자료가 본문과 어긋나면 외부 협업자에게 잘못된 정보가 노출되므로, 한쪽만 변경된 상태로 머지·푸시는 금지합니다. 동기화 영역: **헤더 버전**(`subtitle`의 `vX.Y.Z` + 날짜) · **Phase 산출물 카드** · **마일스톤 표** · **Phase 3 서브 워크스트림 카드** · **크리티컬 패스 사슬** · **병렬화 영역 박스** · **마켓 트랙 항목** · **리스크 매트릭스 + 상세 표** · **갱신 정책 본문**. 단순 typo 수정도 양쪽 모두 적용. CSS·레이아웃 단독 수정만 예외.

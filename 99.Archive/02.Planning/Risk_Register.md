# ⚠️ 리스크 레지스터 (Risk Register / RAID Log)

OkeyMeal MVP(2026-09-30 정식 배포 데드라인)의 리스크·가정·이슈·의존성을 통합 관리하는 RAID 로그입니다. **WBS Task 1.6**의 산출물로, Task 1.1~1.5의 모든 분석 산출물을 종합합니다.

> 📌 **단일 진실 공급원(SSOT) 정책**
> - **리스크 ID(R1~R*)·완화책의 정본은 본 문서**입니다. [`WBS.md`](./WBS.md) §6은 요약(핵심 12건), 본 문서가 전체 상세·소유자·트리거·잔여 위험을 보유합니다.
> - WBS §6 ↔ 본 문서: WBS 측에 ID·리스크명·영향·확률·완화책만 미러링. 본 문서 변경 시 WBS §6도 동일 커밋에서 갱신합니다.
> - **검토 주기**: 매주 일요일 회고 + 마일스톤(M1~M7) 도달 시 게이트 회고에서 일괄 점검.
> - **리스크 등급 변경 시 영향**: 영향·확률·완화책 변경은 본 표 + WBS §6 + (필요 시) Requirements §15 동시 갱신.

## 1. 분류 및 척도

### 1.1 리스크 분류 (Category)
- **TECH**: 기술 리스크 (AI/LLM·OCR·인프라·DB·외부 API)
- **MARKET**: 시장 리스크 (Google Play·경쟁자·사용자 수용)
- **OPS**: 운영 리스크 (인력·일정·비용·운영 안정성)
- **SEC**: 보안·개인정보 리스크 (K-PIPA·시크릿 관리·취약점)
- **LEGAL**: 법무·규제 리스크 (개인정보·데이터 라이선스·심의)
- **PARTNER**: 외부 협력사·점주 의존 리스크

### 1.2 영향도·확률 척도
- **영향도**: 높음(데드라인·시연·법적 책임 직격) / 중간(일정·품질 압축 발생) / 낮음(우회 가능, 일정 영향 미미)
- **발생 확률**: 높음(>50%) / 중간(15~50%) / 낮음(<15%)
- **버퍼**: 일정 버퍼 또는 대체 시나리오 명시. "—"는 즉시 대응(버퍼 불필요).

### 1.3 상태(Status)
- 🟢 **모니터링** (정상 / 트리거 미발생)
- 🟡 **주의** (트리거 임박 / 부분 발생)
- 🔴 **활성** (실제 발생, 대응 진행 중)
- ✅ **종결** (리스크 소멸 또는 결정 완료)

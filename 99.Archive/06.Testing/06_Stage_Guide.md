## 🔍 OkeyMeal 핵심 테스트 영역

테스트 시나리오 작성 시 아래 핵심 기능을 반드시 포함해야 합니다.

| 테스트 영역 | 검증 포인트 | 관련 기능 ID |
|---|---|---|
| **식이 프로필 온보딩** | 22종 알레르기 설정, 비건 레벨, 종교 제약 저장 정확성 | USR-1.1 |
| **듀얼 맵 전환** | 구글맵 ↔ 카카오맵 전환, 안심 신호등 핀 표시 | USR-2.1, USR-2.2 |
| **AI 렌즈 (Fast-Check)** | 메뉴판 OCR 정확도, 개인 프로필 대비 안전/주의/위험 판정, 신뢰도 점수 표시 | USR-3.1, INF-02, INF-04 |
| **QR 점주 알림** | QR 스캔 → 점주 수신까지 3초 이내, 한국어 질문 정확성, AI 대체 레시피 생성 | USR-3.2, OWN-2.1, OWN-2.4 |
| **SOS 핫라인** | 현재 위치 기반 외국어 대응 의료기관 매핑 정확성 | USR-3.4, INF-07 |
| **점주 응답** | 안전/위험/대안 응답 → 사용자 앱 표시 일치성 | OWN-2.2 |
| **비회원 모드** | 로그인 없이 AI 렌즈 기능 사용 가능 여부 | OQ-02 관련 |
| **성능** | AI 렌즈 응답 3초 이내, QR 알림 수신 3초 이내 | INF-05 |

## 📝 주요 문서 템플릿

### Test_Scenario.md

```markdown
| TC-ID | 테스트 항목 | 전제 조건 | 테스트 단계 | 기대 결과 | 실제 결과 | P/F |
|---|---|---|---|---|---|---|
| TC-001 | AI 렌즈 — 땅콩 알레르기 감지 | 프로필에 땅콩 알레르기 설정 | 1. 메뉴판 촬영 2. 분석 요청 | '위험' 신호등 + 한국어 해설 표시 | | |
```

### Bug_Report.md

```markdown
## [BUG-001] 버그 제목

| 항목 | 내용 |
|---|---|
| **발견일** | YYYY-MM-DD |
| **심각도** | Critical / High / Medium / Low |
| **상태** | Open / In Progress / Closed |
| **관련 기능** | 기능 ID |

### 재현 단계
1. 단계 1
2. 단계 2

### 기대 결과 vs 실제 결과

### 스크린샷 / 로그
```

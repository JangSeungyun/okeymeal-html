# 📊 OkeyMeal 공공데이터 및 API 리서치

이 문서는 OkeyMeal v1.5.0의 핵심 기능 구현을 위해 활용되는 공공데이터, 민간 API, 그리고 내부 생성 데이터 자산의 상세 항목을 기술합니다.

---

## 1. 공공데이터 리스트 (Public Data)

### ① 한국관광공사: Tour API 및 다국어 서비스
- **활용 API:** 국문/영문/일문/중문(간체/번체) 관광정보 서비스
- **제공 데이터:** 식당 명칭, 위치(좌표), 다국어 소개, 이미지, 영업시간 등.
- **활용 방안:** 글로벌 관광객 대상의 다국어 식당 DB 구축 및 검색 엔진의 기본 소스로 활용.

### ② 한국관광공사: 무장애 및 특화 관광 정보
- **활용 API:** 무장애 여행 정보, 웰니스 관광정보, 의료 관광정보
- **제공 데이터:** 휠체어 접근성, 편의시설 정보, 건강/치유 특화 장소 정보.
- **활용 방안:** 식이 제약뿐만 아니라 물리적 접근성(Barrier-free)을 고려한 안심 경로 및 힐링 식당 큐레이션.

### ③ 한국관광공사: 빅데이터 및 연관 정보 (Advanced)
- **활용 API:** 관광지별 연관 관광지 정보, 관광빅데이터 정보서비스(DataLab)
- **제공 데이터:** 관광지-식당 간의 연결성 데이터, 지자체별 방문자 추이 및 소비 다양성 지표.
- **활용 방안:** 
    - **연관 정보:** "이 관광지를 방문한 사용자들이 선호한 안심 식당" 추천(Collaborative Filtering).
    - **데이터랩:** B2B 대시보드 구축 시 지역별 외국인 미식 트렌드 분석 리포트의 근거 자료로 활용.

### ④ 식품의약품안전처: 식품안전나라 (RAG 핵심 데이터)
- **활용 API:** 
    - **조리식품의 레시피 DB (COOKRCP01):** 전국 주요 조리 식품의 표준 레시피, 재료 정보 등.
    - **식품영양성분 DB 정보 (I2791):** 가공식품 및 조리 식품의 탄수화물, 단백질, 나트륨 등 영양 성분 상세.
    - **건강기능식품 영양 DB (I0760):** 국내 유통 건강기능식품의 기능성 성분 및 영양 정보.
- **활용 방안:** 
    - **표준 레시피(COOKRCP01):** 점주가 레시피를 직접 입력하지 않은 식당의 경우, AI가 이 DB를 바탕으로 "일반적인 조리 방식에 포함된 알레르기 유발 물질"을 1차 추론함.
    - **영양성분(I2791):** 당뇨, 저염식 등 식이 조절이 필요한 관광객에게 성분 함량 기반의 안심 가이드 제공.
    - **건기식/가공식품(I0760, I2791):** Phase 3의 'AI 렌즈' 기능을 통해 편의점 제품이나 건강기능식품 스캔 시 맞춤형 해설 제공.

## 2. 민간 인프라 API (Private Infrastructure)

### ① Kakao Maps API (주최사 핵심 기술)
- **활용 방안:** 
    - **Dual Map Engine:** 국내 최적화된 상세 지도 및 실시간 길 찾기 제공.
    - **Local POI:** 관광공사 데이터에 없는 실시간 장소 리뷰 및 상세 위치 보정.
    - **Web SDK:** 점주용 관리 페이지 내 식당 위치 설정 및 QR 지도 연동.

### ② Google Maps Platform
- **활용 방안:** 글로벌 관광객에게 익숙한 UX 제공 및 사용자 국가별 언어 설정 자동 연동.

### ③ Google Gemini API (Intelligence)
- **활용 방안:** 4-Layer 데이터 융합 추론, 점주용 한국어 질문 자동 생성, 신뢰도 점수(Confidence Score) 산출.

## 3. 내부 및 점주 생성 데이터 (Internal & Owner Data)

### ① 점주 직접 입력 레시피 (Owner-Generated Recipe)
- **항목:** 메뉴별 고유 식재료, 주방 내 교차 오염 가능 물질, 대체 식재료 정보.
- **활용 방안:** 공공데이터의 시차를 보완하는 **최우선 신뢰 데이터**로 활용.

### ② 사용자 식이 프로필 및 페르소나 로그
- **항목:** 개인별 알레르기/종교/지향적 식이 제약 정보, 방문 및 안심 확인 이력.
- **활용 방안:** "Fast-Check" 시 개인 맞춤형 신호등 표시 및 재방문 추천 알고리즘 학습.

---

## 4. 데이터 융합 파이프라인 (Data Fusion Pipeline)
1.  **데이터 수집:** Tour API(기본 정보) + 식약처(성분) + 카카오맵(상세 위치).
2.  **데이터 정제:** Gemini AI가 중복 정보를 제거하고 표준 데이터 스키마로 변환.
3.  **데이터 보강:** 점주 관리 페이지를 통해 실제 레시피 정보 결합.
4.  **최종 서비스:** 사용자 프로필과 대조하여 개인화된 안심 정보 출력.

# 📋 요구사항 정의서 (Requirements)

본 문서는 OkeyMeal 프로젝트의 1단계 분석 전략을 바탕으로 개발에 필요한 **기능적 요구사항(Functional Requirements, FR)** 과 **비기능적 요구사항(Non-Functional Requirements, NFR)** 을 상세하게 명세합니다.

---

## 1. B2C 기능 요구사항 (관광객용 앱)

| ID | 기능명 | 상세 내용 | 우선순위 |
| :--- | :--- | :--- | :---: |
| **FR-C01** | **건강 프로필 설정 (Onboarding)** | <ul><li>회원가입 직후 필수 온보딩(Onboarding) 프로세스로 구성</li><li>기저질환(제2형 당뇨, 고혈압) 및 알레르기 유발 물질 21종(식약처 기준) 체크리스트 UI 제공</li><li>할랄, 비건, 글루텐프리 등 추가 식이 성향 선택 기능</li><li>선호 언어 설정(영어, 일본어, 중국어 간/번체 등 지원)</li></ul> | 필수 (High) |
| **FR-C02** | **안심 식당 탐색 및 필터링** | <ul><li>한국관광공사 TourAPI의 '무장애 여행' 데이터 및 주변 식당 리스트 맵핑</li><li>사용자 프로필 기반 위험도 시각화(안전:녹색, 주의:황색, 위험:적색) 필터 제공</li><li>정렬 기준: 거리순, 나와 같은 기저질환자/식이제한자 방문순, 전체 평점순</li></ul> | 필수 (High) |
| **FR-C03** | **AI 렌즈 (메뉴판 스캐너)** | <ul><li>스마트폰 카메라 구동 시 실시간 로딩 딜레이 처리(스켈레톤 UI 제공)</li><li>다국어 실시간 AR 오버레이 번역 및 성분 분석 결과에 따른 적색 경고 노출</li><li>**Fallback 시나리오**: 빛반사 등으로 OCR 인식 실패 시, 사용자가 수동 텍스트 입력 또는 음성 입력으로 메뉴를 검색할 수 있는 대체 기능 제공</li></ul> | 필수 (High) |
| **FR-C04** | **식후 맞춤형 산책로 추천** | <ul><li>식사 완료(결제) 후, 현재 식당 반경 2km 이내의 걷기 코스 추천 (TourAPI 연동)</li><li>당뇨 환자 등의 식후 혈당 스파이크 방지 목적을 강조하는 푸시 알림 발송</li><li>코스별 소요 시간(예: 30분 코스, 1시간 코스) 명시 및 구글맵 연동 도보 네비게이션 제공</li></ul> | 권장 (Medium) |
| **FR-C05** | **SOS 응급 상황 대처** | <ul><li>알레르기 쇼크(아나필락시스) 등 발생 시 최단 거리 응급 의료기관 안내 (구글 맵스 연동)</li><li>다국어 헬프 카드("저는 [땅콩] 알레르기 환자입니다. 119를 불러주세요") 전체화면 노출 및 음성 읽어주기(TTS) 지원</li></ul> | 필수 (High) |
| **FR-C06** | **오프라인 헬프 모드 (신규)** | <ul><li>여행 중 통신망(데이터/와이파이) 접속 불량 시를 대비하여, 기기 로컬 캐시에 저장된 프로필을 바탕으로 기본 알레르기 설명 화면 및 SOS 헬프 카드를 오프라인에서도 구동되도록 지원</li></ul> | 필수 (High) |
| **FR-C07** | **동일 식성 커뮤니티 리뷰 (신규)** | <ul><li>식당 리뷰 조회 시 작성자의 '건강 프로필 뱃지(예: 저당 식단, 땅콩 알레르기)' 노출</li><li>나와 비슷한 식성을 가진 사용자의 리뷰만 모아볼 수 있는 스마트 필터링 제공</li><li>유용한 리뷰 작성자 및 식후 산책 목표 달성자에게 게이미피케이션(뱃지/포인트) 보상 지급</li></ul> | 선택 (Low) |

## 2. B2B 기능 요구사항 (소상공인 식당 점주용)

| ID | 기능명 | 상세 내용 | 우선순위 |
| :--- | :--- | :--- | :---: |
| **FR-B01** | **스마트 오더 뷰어 (QR 기반)** | <ul><li>별도 앱 설치 불필요. 관광객의 스마트폰 화면에 생성된 QR을 점주가 스캔하여 접속</li><li>관광객의 주문 내역 및 주의사항(예: "견과류 절대 금지")이 가장 직관적인 한국어로 크게 번역되어 출력됨</li></ul> | 필수 (High) |
| **FR-B02** | **원터치 다국어 퀵 리플라이 (신규)** | <ul><li>점주가 주문 확인 후 브라우저 화면에서 "견과류 빼 드릴게요", "재료 소진", "조금 덜 짜게 해드릴게요" 등의 준비된 템플릿 버튼(원터치)을 클릭</li><li>해당 메시지가 관광객의 설정 언어로 즉시 번역되어 앱의 푸시 알림으로 전송됨 (점주의 외국어 소통 부담 완벽 해소)</li></ul> | 필수 (High) |

## 3. 비기능적 요구사항 및 보안 (Non-Functional Requirements)

| ID | 분류 | 상세 내용 |
| :--- | :--- | :--- |
| **NFR-S01** | **보안 (Quishing 防禦)** | QR 피싱(Quishing) 철저 방어. 공식 도메인(`owner.okeymeal.kr`) 강제 및 JWT/HMAC 기반의 5분 만료 동적 서명 활용. 접속 시 공식 인증 녹색 뱃지 애니메이션 노출. |
| **NFR-S02** | **보안 (Privacy)** | GDPR 및 개인정보보호법 준수. 사용자의 질환/알레르기 정보 등 '민감 건강 정보(PHI)'는 DB 저장 시 양방향 암호화(AES-256) 처리 필수 적용. |
| **NFR-P01** | **성능 (Performance)** | AI 렌즈의 이미지 스캔부터 위험도 분석 렌더링까지의 E2E(End-to-End) 지연 시간을 최대 3초 이내로 보장. |
| **NFR-R01** | **가용성 (Reliability)** | 핵심 서버 가용성(Uptime) 99.9% 보장 및 오프라인 모드 전환 안정성 유지. |
| **NFR-U01** | **사용성 (Usability)** | 점주용 뷰어 웹페이지는 60대 이상 고령층을 고려하여 최소 24px 이상 명조/고딕 혼용 폰트 및 고대비(High Contrast) 배색 적용. |

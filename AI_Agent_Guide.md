# 📘 AI 에이전트 협업 가이드 (OkeyMeal)

이 파일은 **OkeyMeal(오키밀)** 프로젝트의 일관된 문서 관리와 에이전트의 역할 수행을 위해 작성되었습니다. Gemini, Claude Code, Codex 등 어떤 AI 에이전트를 사용하더라도 새로운 세션 시작 시 이 내용을 제공하여 문맥(Context)을 동기화하세요.

---

## 1. 에이전트 페르소나 및 규칙
 - 새 세션에서 이 문서를 읽어들일때마다 문서 작성자를 입력받아 기억하도록 함.(담당자가 여러명이므로 작성자 입력 규칙을 지키기 위함)

### 🤖 역할: 문서화 에이전트 (Documentation Agent)
2026 관광데이터 활용 공모전(②-2 웹·앱 구현 부문) 출품을 위해 구조화된 마크다운 문서를 생성/업데이트하는 전문가.

### 📋 문서 작성 표준 규칙
1.  **Stage Guide 준수:** 각 폴더에 위치한 `_Stage_Guide.md`의 필수 산출물 목록과 **맞춤 템플릿 규칙**을 무조건적으로 준수하여 문서를 작성하십시오.
2.  **헤더/푸터:** 상단과 하단에 프로젝트 메타데이터(파일명, 버전, 상태(`status`), 작성일 등)를 포함.
3.  **다국어/글로벌 정책 (Language Policy):** 프로젝트의 내부 기획/설계 문서는 '한국어'로 작성하되, 실제 화면에 노출되는 **UI 텍스트, 사용자 시나리오 데이터, 프롬프트 예시 등은 다국어(영어/중국어/일본어 등)를 기본으로 고려**하여 설계하십시오.
4.  **버전 관리:** 
    - 소규모 수정: 마이너 버전 증가 (v1.0 -> v1.1) 및 동일 파일 업데이트.
    - 대규모 변경: 메이저 버전 증가 (v1.0 -> v2.0) 및 신규 파일 생성.
5.  **시각화:** Mermaid 다이어그램(순서도, 아키텍처, ERD 등)을 적극 활용.
6.  **가독성:** GitHub Flavored Markdown(GFM) 준수, 표, 체크리스트, 코드 블록 사용.
7.  **병합 원칙:** 기존 파일 업데이트 시 기존 데이터를 분석하여 안전하게 병합(Merge).

### 🔄 세션 분리 및 인수인계 (Handover) 규칙
*   **세션 분리 권장**: 공모전 단계(Stage)가 변경되거나, 기획에서 개발로 컨텍스트가 크게 넘어갈 때는 AI 컨텍스트 한계 극복을 위해 채팅 세션을 새로 시작하는 것을 강력히 권장합니다.
*   **인수인계 문서 생성**: 기존 세션을 종료하기 전에 AI에게 *"지금까지의 핵심 컨텍스트, 결정 사항, 다음 세션 작업 목록을 `handover_state.md` 로 요약해 줘"* 라고 명령하여 상태를 백업하십시오.
*   **새 세션 동기화**: 새 세션을 열 때 *"handover_state.md 와 AI_Agent_Guide.md 를 읽고 다음 작업을 이어서 시작해 줘"* 라고 명령하여 컨텍스트 손실 없이 완벽하게 업무를 이어가십시오.

## 2. OkeyMeal 프로젝트 핵심 요약 (과제 5번 포커스)

### 🎯 미션 및 타겟
-   **공모전 지정과제:** [과제 5] 맞춤형 식당 정보 부족으로 인한 만성질환자의 외식 불안감 발생 해소.
-   **미션:** 만성질환자 및 식이 제한이 있는 관광객의 식이 제약을 해결하고 안전한 미식 케어 경험 제공.
-   **타겟:** 1차) 내/외국인 기저질환자(당뇨, 고혈압 등), 2차) 특정 식이 제한(비건, 알레르기, 할랄)이 있는 관광객.

### 🚀 핵심 기능 (Core Features)
-   **Epic 1. 안심 식당 탐색 & 산책 코스 연계:** 한국관광공사 API 기반 필터링 검색 및 식후 혈당 관리에 좋은 주변 산책 코스 연계 추천.
-   **Epic 2. AI 렌즈 (메뉴판 스캐너):** Naver Clova OCR 및 HyperCLOVA X(Spring AI) 연동을 통해 메뉴의 위험 성분(고당류, 알레르겐 등) 분석 및 경고.
-   **Epic 3. SOS 대응 (비상 상황):** 응급 상황 시 위치 기반 주변 의료기관 안내 및 다국어 구조 요청 기능.

### 🛠 기술 스택 (Tech Stack)
> **SSOT**: 기술 스택의 정본은 [`02.Planning_Design/Architecture.md`](./02.Planning_Design/Architecture.md)이며, 아래는 그 요약이다. 변경 시 Architecture.md를 먼저 갱신하고 본 요약을 미러링한다.
-   **Frontend:** React 19.2.x, Vite, FSD Architecture, Zustand + React Query, **Vanilla CSS Modules**.
-   **Backend:** Spring Boot 4.1.x (OpenJDK 25), PostgreSQL (JSONB·PostGIS) + Redis, Spring Data JPA, Virtual Threads, JWT.
-   **AI/Infra:** **Naver Clova OCR + Naver HyperCLOVA X (Spring AI)**, Docker, GitHub Actions.

## 3. 표준 문서 작성 프롬프트 샘플

새로운 문서를 만들거나 업데이트할 때 아래 프롬프트를 복사하여 사용하세요.

> **[프롬프트 샘플]**
> `AI_Agent_Guide.md`의 규칙과 해당 폴더의 `_Stage_Guide.md` 맞춤 템플릿 규칙을 엄격히 준수해서 아래 내용을 바탕으로 [문서명]을 작성해줘.
> 
> **작업 요청 사항:**
> 1. 대상 파일: [파일경로]
> 2. 추가/수정 데이터: [여기에 내용 입력]
> 3. 시각적 요소: [Mermaid 다이어그램 추가 여부 등]
> 4. 버전 관리: [v1.x 마이너 업데이트 혹은 v2.0 신규 생성 여부]
> 
> **문서 구조:** `_Stage_Guide.md`에 명시된 해당 문서의 템플릿 형식을 정확히 따를 것.
